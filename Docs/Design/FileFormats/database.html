File Format Documentation

Database

Introduction

From CodeSnip version 5 the program stores its database as a series of binary files in the program's database directory.

When using CodeSnip in standard mode each logged on user has a separate database directory, which can be changed by the user. In portable mode the database is located in a fixed sub-directory of the program directory.

At present there is only a single version of the binary database format – version 7.

Database versions 1 to 6 were XML based and are documented elsewhere. CodeSnip 5 retains the ability to read, but not write, all the XML formats.

Encoding

These files are stored in binary format. Text included in within the binary data is stored in UTF‑8 format.

File Format

There are two types of file in use. There is a single "master" database file and a separate file for each snippet. the "master" and "snippet" files are described separately in the sub-sections below.

There are several "data types" used in the files, which are as follows:

Byte
Unsigned 8 bit integer value.
Word
Unsigned 16 bit integer value.
LongWord
Unsigned 32 bit integer value.
SizedString
This is a variable length UTF‑8 string that is preceded by its length as a Word.
SizedLongString
This is a variable length UTF‑8 string that is preceded by its length as a LongWord.
StringList
This is a representation of a list of SizedString values. It is a sequence of zero or more SizedString values preceded by the number of strings in the list as a LongWord.
FixedString[X]
This is a chunk of UTF‑8 text whose length is specified by X.
FixedBytes[X]
This is a fixed size chunk of Byte values whose size is specified by X.
Date
This is a UTC date in ISO‑8601 format written as a SizedString value. The string will always have the format YYYY‑MM‑DD"T"HH:MM:SS.XXX"Z" or YYYY‑MM‑DD"T"HH:MM:SS,XXX"Z".

Master File Format

There is a single master database file in the database. It is named csdb.master.dat.

The file consists of a number of fields in a fixed order, as follows:

Watermark (FixedString[44])
Identifies the file as a master database file. The field always has the value CSDB‑MASTER‑0562C0E6F20B4C2F945D1FDEC6393C4C.
FileVersion (FixedString[4])
The version of the database as a four character string of digits. At present only version 7 is supported and the field always has the value 0007.
LastModificationDate (Date)
Date the file was last modified.
SnippetCount (LongWord)
The number of snippets in the database, followed by SnippetCount copies of the following fields:
ID (SizedString)
Text representation the snippet's unique identifier.
LastModificationDate (Date)
Date the snippet was last modified.
TagCount (LongWord)
The number of tags in the database, followed by TagCount copies of the following fields:
TagName (SizedString)
The name of the tag as a string.
Reserved (LongWord)
A 32 bit field reserved for future use. Must be 0.

Snippet File Format

There is one snippet file for each snippet in the database. The files are named csdb.snippet.ID.dat where ID is a string representation of the snippet's ID.

The file consists of a number of required fields in a pre-determined order, followed by the snippet's property data in any order, some of which are optional.

Fixed Fields

The fixed fields are as follows. They must appear in the given order.

Watermark (FixedString[45])
Identifies the file as a snippet file. The field always has the value CSDB‑SNIPPET‑0562C0E6F20B4C2F945D1FDEC6393C4C.
FileVersion (FixedString[4])
The version of the database as a four character string of digits. At present only version 7 is supported and the field always has the value 0007.

Property Data

Following the fixed fields comes a list of property data. Properties can appear in any order. Some properties must be written – they are noted below. Others are optional and won't be written if they have their default values. Default values are noted for optionally written properties.

The property data is organised as follows, one record for each property:

PropertyID (Byte)
A unique code that identifies the property. Property IDs must be in the range 0x00..0xFE. The special value 0xFF is used to terminate the sequence of property records.
PropertyData (type is property dependent)
Data describing the property. The data is property dependant, as follows:
ID property
A SizedString value that is a text representation of the snippet's ID. Must be non-empty and a valid ID string. Required.
Title property
A SizedString value containing the snippet's title in plain text. Must be non-empty. Required.
Description property
A SizedLongString value containing the snippet's description in REML mark-up. REML v4 is supported. Default: no description.
SourceCode property
A SizedLongString value containing the snippet's source code in plain text. Must be non-empty. Required.
LanguageID property
A SizedString value that is a text representation of the snippet's language ID. Default: default language ID.
Modified property
A Date value that is the date the snippet was last modified. Must be valid UTC date-time string. Required.
Created property
A Date value that is the date the snippet was created. must be valid UTC date-time string. Required.
RequiredModules property
A StringList value containing a list of any required module names, each of which is in plain text. Default: no required modules.
RequiredSnippets property
A StringList value containing a list of string representations of the IDs of any required snippets. Any ID strings must be valid snippet IDs. Default: no required snippets.
XRefs property
A StringList value containing a list of string representations of the IDs of any cross referenced snippets. Any ID strings must be valid snippet IDs. Default: no cross referenced snippets.
Notes property
A SizedLongString value containing the notes in REML mark-up. REML v4 is supported. Default: no notes.
Kind property
A Byte value containing a code that specifies the snippet's kind. Default: freeform.
CompileResults property
There are two consecutive fields:
  1. A StringList containing the ID strings of compilers with which the snippet compiles successfully.
  2. A StringList containing the ID strings of compilers with which the snippet fails to compile.
Default: no compile results.
Tags property
A StringList value containing a list of names of the snippet's tags, each in plain text. Each tag name must be made up of valid tag characters. Default: no tags.
